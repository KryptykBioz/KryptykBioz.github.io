flagship

ANNA_AI

/BASE/cognitive_loop_manager.py

A modular, continuously running agentic AI framework built on Ollama. Separates internal thought from spoken output via a two-stage cognitive architecture — reactive, proactive, reflective, and action-driven modes — with a secondary AI “speak judge” that vetoes repetitive or low-value responses before TTS fires.

  • Four-tier encrypted memory (AES-256-GCM, atomic writes, cosine similarity search)
  • Hot-reloadable core & tool modules — drop in a directory, no restart required
  • Discord / Twitch / YouTube live chat ingestion as raw cognitive events
  • Custom Tkinter GUI: config, controls, file injection, tool status, color-coded logs
  • Optional uncensored response model + universal kill-phrase failsafe
Python Ollama PyTorch AsyncIO AES-256-GCM Tkinter

Setup Guide

Installation is fully automated. Clone the repo, run the installer, and launch — the script handles everything from the virtual environment to model downloads. Full documentation is in SETUP.md and README.md.

01

Prerequisites

Install the following before running the installer. The installer will detect and report any that are missing.

Python 3.11.xpython.org/downloads/release/python-3119/ — 64-bit, Windows only
Gitgit-scm.com/download/win — required for tool module installs
Ollamaollama.ai/download — runs all local LLM inference

[Note] RTX 50-series (Blackwell / sm_120) users: download the blackwell_sm120_wheels repository and place it in the project root before running the installer. The installer will detect your GPU and handle wheel assembly automatically.

02

Clone & run the installer

Clone the repo, then double-click INSTALL.bat (or run it from a terminal). The installer walks through every step interactively.

git clone https://github.com/KryptykBioz/Anna_AI.git
cd Anna_AI
INSTALL.bat

The installer covers: system checks → virtual environment → Python dependencies → PyTorch & GPU packages → Ollama model downloads → .env configuration → directory scaffold → optional components → verification.

03

Configure agent identity

The installer prompts for a name and how the agent addresses you, writing both into .env. Further customization lives in two files post-install:

personality/bot_info.pyAgent name, username, model assignments
personality/config.jsonAdvanced Ollama, voice, and memory settings
.envAPI tokens (Discord, Twitch, YouTube) and Ollama tuning overrides

[Note] Core Ollama models pulled during install: gemma4:latest (~7 GB) for all reasoning loops, and nomic-embed-text (~275 MB) for memory embeddings. Optional vision models (qwen3-vl:8b) can be pulled during install or at any time with ollama pull.

04

(Optional) Install tool modules

The installer presents a numbered list of available tool modules and clones your selections from GitHub into BASE/tools/installed/. Tools can also be added at any time manually. 

git clone https://github.com/KryptykBioz/ANNA-tool-<name> BASE/tools/installed/<name>

Available modules include: web search, Discord/Twitch/YouTube chat, Minecraft, image generation, screenshot vision, calendar, reminders, notes, system info, and more.

05

(Optional) Virtual audio cables

Required for avatar lip-sync — routes TTS speech through a virtual device so the avatar animation system can read the audio signal. Not needed for text-only operation. The installer can open the download page and walk through the manual steps.

VB-Audio Cablevb-audio.com/Cable/ — install as Administrator, then restart
Tala CablesOptional, for multi-agent audio routing

After install, configure vb_cable_name in personality/bot_info.py to match the exact Windows device name (e.g. "CABLE Input").

06

Launch

Double-click START.bat or run it from a terminal. It starts Ollama, activates the virtual environment, sets CUDA paths, and launches the GUI. The cognitive loop begins automatically.

START.bat

[Note] If Ollama is not already running, START.bat starts it automatically and waits for initialization before launching the agent.

Troubleshooting

Common issues pulled directly from build & runtime logs.

$ SystemError: PY_SSIZE_T_CLEAN macro must be defined

Cause: The PyAudio wheel was built without the required source patch for Python 3.11+.

Fix: Re-apply the PY_SSIZE_T_CLEAN patch to _portaudiomodule.c, clean the build directory (build/, dist/, *.egg-info), and rebuild.

$ torch.cuda.is_available() returns False after install

Cause: CUDA/cuDNN runtime DLLs were not copied into torch/lib after installing a custom-built wheel.

Fix:

Copy-Item "$CUDA_PATH\bin\x64\*.dll" ".\venv\Lib\site-packages\torch\lib" -Force
Copy-Item "$CUDA_PATH\bin\cudnn64_9.dll" ".\venv\Lib\site-packages\torch\lib" -Force
$ SYCL: feature test compile failed during PyTorch build

Cause: CMake detected Intel oneAPI/SYCL from an Arc GPU present on the same system.

Fix: Set $env:USE_XPU = "0" before configuring the build.

$ 32-bit compiler (HostX86\x86\cl.exe) selected during build

Cause: vcvars64.bat was not initialized before running build commands.

Fix: Run the x64 environment script first, in the same shell:

cmd /c '"...\BuildTools\VC\Auxiliary\Build\vcvars64.bat" && powershell'
$ TTS import fails after installing custom PyTorch

Cause: torchaudio conflicts with the custom PyTorch build.

Fix: Uninstall torchaudio and patch TTS/tts/models/vits.py to make the import optional:

pip uninstall torchaudio -y

# vits.py
try:
    import torchaudio
except ImportError:
    torchaudio = None
$ Encrypted memory files unreadable after moving install

Cause: Tiers 1–3 of the memory system are encrypted with AES-256-GCM using a per-install key stored in personality/memory/.machine_secret.

Fix: Never delete .machine_secret without also deleting the encrypted memory files — doing so permanently locks the data. To reset memory cleanly, delete both together; the agent starts with a blank history on next launch.

$ vswhere.exe returns empty / VS not detected

Cause: Visual Studio is installed but not registered with the VS installer database.

Fix: Skip vswhere entirely and reference vcvars64.bat by its direct install path.

$ A custom tool isn't being picked up by the agent

Cause: Missing or malformed information.json, or the tool directory isn't under BASE/tools/installed/.

Fix: Confirm the directory contains both tool.py (a BaseTool subclass) and information.json with at minimum tool_name, control_variable_name, tool_description, and available_commands. ToolLifecycleManager discovers new tools on next startup or hot reload — no manual registration needed.

Not seeing your issue? Open an issue on the relevant repo — Anna_AI/issues or AI_Agent_Tools/issues.
